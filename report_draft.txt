Tiny Transformer for Next-Token Prediction on the Tiny Shakespeare Corpus

1. Introduction

This project implements a lightweight decoder-only Transformer in PyTorch for next-token prediction on the Tiny Shakespeare corpus. The goal was to build the main components of a Transformer from scratch in a computationally manageable setting while still preserving the core ideas used in modern language models. The implementation includes subword tokenization, fixed-length overlapping training sequences, sinusoidal positional encoding, masked multi-head self-attention, residual connections, RMSNorm, a feed-forward network, and evaluation with validation perplexity.

The assignment emphasizes both implementation and interpretation. Accordingly, this report focuses not only on whether the model trains successfully, but also on what the loss curves and attention maps reveal about the model's behavior. The model is intentionally small, with two Transformer blocks and a hidden size of 128, so the resulting system is not expected to generate highly fluent Shakespearean text. Instead, the objective is to verify that the architecture functions correctly, trains stably, and exhibits meaningful attention structure on a real next-token prediction task.

2. Dataset and Preprocessing

The dataset used in this assignment is the Tiny Shakespeare corpus, a compact text dataset commonly used for language-model experiments. The raw corpus contained 1,115,394 characters. Since the task is next-token prediction, the model is trained to predict the following token at every position in a sequence.

I used a subword-level tokenizer based on Byte Pair Encoding (BPE) rather than character-level tokenization. This choice allows the model to represent frequent text fragments such as common prefixes, suffixes, and short words more efficiently than pure character tokenization. To remain within the assignment constraints, the vocabulary size was capped at 500 tokens. After tokenization, the corpus contained 500,959 tokens.

After tokenization, the corpus was split into overlapping fixed-length windows of 50 tokens using a stride of 2. For each sequence, the input consists of the first N tokens and the target is the same sequence shifted one step to the left. This directly matches the autoregressive next-token prediction objective. The resulting dataset was divided into 80% training data and 20% validation data, with no separate test set, following the assignment instructions.

3. Model Architecture

The model is a small decoder-only Transformer language model implemented in PyTorch. Its main configuration is summarized below:

- Vocabulary size: 500
- Context length: 50
- Embedding dimension: 128
- Number of attention heads: 4
- Number of Transformer blocks: 2
- Feed-forward expansion multiplier: 4
- Dropout: 0.1
- Learning rate: 3e-4
- Weight decay: 1e-2
- Batch size: 64
- Number of epochs: 6
- Trainable parameters: 524,660

The model first maps token IDs to dense vectors using nn.Embedding. Sinusoidal positional encodings are then added to the token embeddings so that the model can distinguish token order. This is essential because self-attention alone is permutation-invariant and would otherwise have no notion of sequence position.

Each Transformer block uses a pre-norm structure. RMSNorm is applied before both the self-attention sublayer and the feed-forward sublayer. The attention mechanism is implemented manually using learned projections for queries, keys, and values, followed by scaled dot-product attention. A causal mask is applied so that each token can only attend to itself and earlier tokens, preventing information leakage from future positions. Residual connections are used after both the attention and feed-forward sublayers.

The feed-forward network expands the hidden dimension by a factor of four before projecting back to the model dimension. After the stacked Transformer blocks, a final RMSNorm layer and a linear projection produce logits over the vocabulary for each token position.

This design follows the core Transformer recipe while remaining small enough to train quickly on limited hardware. The final model has 524,660 trainable parameters, so it is large enough to learn nontrivial sequence patterns but still lightweight enough for a short Colab GPU run.

4. Training Setup

The model was trained using cross-entropy loss for next-token prediction. I used the AdamW optimizer with learning rate 3e-4 and weight decay 1e-2. Training was run for 6 epochs. To improve reproducibility, random seeds were fixed for Python, NumPy, and PyTorch.

The main evaluation metric was validation perplexity, computed as:

Perplexity = exp(validation cross-entropy loss)

Perplexity is a standard language modeling metric and provides an interpretable measure of how uncertain the model is when predicting the next token. Lower perplexity indicates better predictive performance.

5. Results

5.1 Training and Validation Loss

Figure 1 shows the training and validation loss curves across epochs. The training loss decreased from 3.8127 after the first epoch to 2.9603 after the sixth epoch, a reduction of 0.8524. The validation loss also improved, decreasing from 3.7697 to 3.5964 over the same period. This shows that the model learned useful structure from the corpus rather than only memorizing the training batches.

The training loss decreased more quickly than the validation loss, so a generalization gap appeared by the end of training. However, the validation loss continued to decline slightly through epoch 6 rather than increasing, which suggests that the model was still learning useful held-out patterns and had not entered severe overfitting within this short training run. The slower validation improvement is expected for a small Transformer trained from scratch on a compact corpus.

5.2 Validation Perplexity

The final validation loss was 3.5964, corresponding to a validation perplexity of 36.4671. Perplexity improved from 43.37 after epoch 1 to 36.47 after epoch 6. This means the model became less uncertain on held-out next-token prediction as training progressed. The absolute perplexity is still high compared with large pretrained language models, but it is reasonable for a two-layer Transformer trained from scratch with a vocabulary capped at 500 tokens.

5.3 Attention Visualization

Figures 3 and 4 show attention heatmaps from sample validation sequences. Figure 3 focuses on one attention head, while Figure 4 compares multiple heads from the first Transformer layer. Additional heatmaps compare different heads and layers. The heatmaps use query tokens on the vertical axis and key/attended tokens on the horizontal axis, so diagonal or near-diagonal mass means the model is attending heavily to the current or recent previous tokens.

The attention visualizations are useful because they are not uniformly diffuse: the model assigns stronger weights to particular prior positions. The clearest pattern is local or diagonal attention, which fits the next-token task because nearby words, punctuation, and line breaks often provide the strongest clues in Shakespeare-style dialogue. The multi-head grid shows that attention should be interpreted head by head rather than as a single global pattern; different heads can emphasize different previous positions.

5.4 Qualitative Generation and Learning-Rate Comparison

The sample generations provide a qualitative check on what the trained model learned. For the prompt "ROMEO:", the model generated text with Shakespeare-like formatting and line breaks, including phrases such as "ROMEO: traitor" and later references to "the Tower." For the prompt "JULIET:", it produced dialogue-like text with speaker transitions such as "GLOUCESTER:". The generated text is not fully coherent, but it does show that the model learned local formatting patterns, speaker-name conventions, and short phrase structure from the corpus.

I also ran a lightweight learning-rate comparison on a smaller subset using 1e-4 and 3e-4. After two epochs, 1e-4 reached a validation loss of 6.2827 and validation perplexity of 535.26, while 3e-4 reached a validation loss of 6.1202 and validation perplexity of 454.98. On this small comparison, 3e-4 learned faster and achieved the better validation result, which supports using 3e-4 for the main training run.

6. Discussion

6.1 Observed Attention Patterns

The attention maps suggest that the model relies heavily on local context, which is consistent with the limited context length and the structure of Shakespearean dialogue. Nearby words and punctuation often determine the next token, so diagonal attention is a natural learned behavior. The attention-evolution plot compares an epoch-1 checkpoint with the final model and shows how training changes the internal attention pattern over time. This comparison is important because it connects the quantitative loss reduction to a qualitative change in model behavior.

6.2 Hyperparameters and Stability

Among the hyperparameters, context length, learning rate, and hidden size are likely the most influential for training stability and model quality. The context length determines how much prior information the model can use, but it also affects computational cost because attention scales quadratically with sequence length. The learning-rate comparison showed that 3e-4 outperformed 1e-4 on the small subset after two epochs, reaching lower validation loss and lower perplexity. The hidden size and number of layers determine model capacity, but increasing them raises both memory use and runtime.

In this implementation, the combination of 2 layers, hidden size 128, 4 heads, and 6 epochs provided a reasonable balance between expressiveness and efficiency. The model is large enough to show meaningful learning behavior but still small enough to train within assignment constraints.

6.3 Role of Positional Encoding

Positional encoding is essential in this model. Without positional information, self-attention would treat the input as an unordered set of tokens, making next-token prediction much more difficult. Word order is central to language, especially in sequence modeling tasks. The sinusoidal positional encoding used here provides a lightweight way to inject position information without introducing many additional learnable parameters.

6.4 Runtime and Memory Bottlenecks

The main computational bottleneck is the self-attention operation, whose memory and time costs grow with the square of the sequence length. In the completed run, each epoch took about 32.70 seconds on average, and peak GPU memory was about 122.7 MB. This is manageable for the current context length of 50 and hidden size of 128, but increasing context length would quickly raise the size of the attention matrices. Batch size and embedding dimension also contribute significantly to memory usage, especially during training when gradients must be stored.

Because of these constraints, the assignment's recommendation to keep the model small is well justified. A modest architecture still reveals the essential mechanics of the Transformer without making training prohibitively expensive.

6.5 Limitations

This model has several limitations. First, it is trained on a very small corpus, so its generation quality is inherently constrained. Second, the vocabulary cap of 500 reduces representation richness, though it is appropriate for a lightweight assignment. Third, the model was trained for only a small number of epochs, so some attention patterns may still be underdeveloped. Finally, the architecture is intentionally simple and does not include more advanced design choices found in larger modern language models.

7. Conclusion

This project successfully implemented a lightweight Transformer language model from scratch for next-token prediction on the Tiny Shakespeare corpus. The final system included subword tokenization, overlapping training sequences, sinusoidal positional encoding, masked multi-head self-attention, residual connections, RMSNorm, and a feed-forward network. Across six epochs, training loss decreased from 3.8127 to 2.9603, validation loss decreased from 3.7697 to 3.5964, and validation perplexity improved from 43.37 to 36.47. These results show that the model learned meaningful structure from the corpus while remaining compact at 524,660 trainable parameters.

The attention visualizations and sample generations were particularly useful for interpreting the model's behavior. The generations were not fully coherent, but they reproduced local Shakespeare-like formatting, speaker labels, and short phrase structure. Overall, the experiment provided a clear and practical demonstration of how Transformer components work together in an autoregressive language modeling task.
